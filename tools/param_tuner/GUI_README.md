# ESP32-S3 Parameter Tuner GUI

ESP32-S3マイクロマウス用のパラメータ調整GUIアプリケーションです。

## 概要

このアプリケーションは、既存のCLIベースのパラメータ更新ツール(`tx_term.js`)をElectronベースのGUIアプリケーションに変換したものです。

## 機能

- **シリアルポート自動検出**: 利用可能なUSBシリアルポートを自動的に検出して表示
- **プロファイル管理**: モード別に整理されたYAMLファイルの管理
- **個別/一括送信**: 個別ファイルまたは全ファイルをESP32-S3に送信
- **リアルタイムログ**: シリアル通信のログをリアルタイムで表示
- **モダンUI**: 使いやすいグラフィカルインターフェース

## インストール

### 前提条件

- Node.js (v14以上推奨)
- npm

### セットアップ

```bash
cd tools/param_tuner
npm install
```

## 起動方法

### 方法1: 起動スクリプトを使用

```bash
./start_gui.sh
```

### 方法2: npm コマンドで起動

```bash
npm start
```

## 使い方

### 1. シリアルポート接続

1. 「ポート検索」ボタンをクリックして利用可能なポートを検索
2. ドロップダウンリストからESP32-S3が接続されているポートを選択
3. 「接続」ボタンをクリックして接続

### 2. モード選択

- モード選択ドロップダウンから使用するモードを選択(例: `hf`)
- 選択すると自動的にそのモードのプロファイルファイルが読み込まれます

### 3. パラメータ送信

#### 個別ファイル送信
- ファイル一覧から送信したいファイルの「送信」ボタンをクリック

#### 全ファイル一括送信
- 「全て送信」ボタンをクリックすると、プロファイル固有ファイルとシステムファイルを順番に送信

### 4. ログ確認

- 右側のログエリアにシリアル通信の内容がリアルタイムで表示されます
- 「クリア」ボタンでログをクリア可能

## 通信プロトコル

パラメータ送信時のフォーマット:
```
<ファイル名>@<データ>
```

### YAMLファイルの場合
- YAMLを読み込み、JSON形式に変換して送信
- ファイル名の拡張子を`.txt`または指定モード名に変更

### Mazeファイルの場合
- カンマ区切りのデータを読み込み
- 行列転置処理を実施
- `maze.txt`として送信

## ボーレート

- デフォルト: **3,000,000 bps**
- 変更する場合は `gui/main.js` の `baudRate` を編集してください

## トラブルシューティング

### ポートが検出されない
- ESP32-S3が正しく接続されているか確認
- USBケーブルがデータ転送対応か確認
- デバイスドライバが正しくインストールされているか確認

### 接続エラー
- 他のアプリケーション(Arduino IDE、シリアルモニタなど)でポートを使用していないか確認
- ポートのアクセス権限を確認(`/dev/ttyUSB*`, `/dev/ttyACM*`など)

### Electronが起動しない
```bash
# node_modulesを削除して再インストール
rm -rf node_modules package-lock.json
npm install
```

## 開発

### デバッグ
- アプリケーション起動時に開発者ツールが自動的に開きます
- `gui/main.js` の以下の行をコメントアウトすることで無効化できます:
```javascript
mainWindow.webContents.openDevTools();
```

### ビルド
将来的にスタンドアロンアプリケーションとしてビルドする場合は、`electron-builder`などを使用してください。
